# Release Checklist (A1)

> 目标：每次发布前 5-10 分钟完成一次可重复、可验证检查，避免回归。

## 0) 环境前置
- [ ] 当前分支为 `main`
- [ ] 本地工作区干净（`git status` 无非预期改动）
- [ ] `database.sqlite` 不纳入提交

## 1) 本地质量门禁（必须通过）
- [ ] `npm run lint`
- [ ] `npm run build`
- [ ] `npm run test:smoke`（如本地已启动服务）

## 2) 关键功能回归（手工）

### 2.1 账号与档案
- [ ] 注册/登录正常
- [ ] 注册时可上传头像并保存
- [ ] 设置页可修改头像，保存后顶部/总览同步显示

### 2.2 多孩子链路
- [ ] 新增第2个孩子
- [ ] 顶部“切换孩子”按钮首屏可用
- [ ] 切换后：总览/生长/计划/待办/采购数据上下文一致

### 2.3 总览与交互
- [ ] 总览页渲染正常（无重复日历）
- [ ] 右上头像显示正常（有图/无图占位都正确）
- [ ] 导航顺序正确：总览/生长记录/一周计划/备忘待办/采购清单

### 2.4 导出能力
- [ ] CSV 导出：growth/todos 正常下载
- [ ] PDF 导出可打开且版式正常

## 3) 移动端适配（重点视口）
- [ ] 390x844（iPhone）顶部不拥挤、按钮可点
- [ ] 360x800（Android）导航可滑动、文字不截断
- [ ] 模块卡片无明显溢出/重叠

## 4) 提交与发布
- [ ] 文档已同步：`README.md` / `DEVELOPMENT_PROGRESS.md` / `CHANGELOG.md`
- [ ] commit message 清晰
- [ ] push 到 `main`
- [ ] GitHub Actions `docker-publish` 通过（smoke + docker）

## 5) 发布后抽检
- [ ] 打开首页可登录
- [ ] 随机切换孩子 1 次并验证上下文正确
- [ ] 头像仍可显示（缓存刷新后正常）

## 6) 回滚预案（如出现故障）
- [ ] 记录失败 commit
- [ ] `git revert <commit>`
- [ ] 重新触发发布并复测 1)~3)
