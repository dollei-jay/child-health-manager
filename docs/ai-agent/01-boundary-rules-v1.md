# 01-boundary-rules-v1

> 项目：child-health-manager
> 阶段：Phase 1 / P1-T1（正式版）
> 版本：v1
> 日期：2026-03-21

## 1. 定位
本系统是“AI 驱动的儿童健康管理智能体”，不是普通问答聊天机器人。

目标：
1. 认知层：理解历史数据与当前上下文（孩子/时间段/执行状态）
2. 决策层：输出可执行计划、采购建议、当日/周建议
3. 执行层：在权限内自动写库、回执、可撤销

---

## 2. 三层架构边界

## 2.1 认知层（Read）
- 读取范围：档案、身高体重、生长趋势、待办/计划/采购、复盘、提醒
- 输出要求：回答必须可追溯到已读取数据
- 禁止：凭空假设医疗事实

## 2.2 决策层（Plan）
- 生成内容：下周计划、采购清单、今日/本周/下周建议
- 输出结构：建议内容 + 依据 + 优先级 + 执行动作
- 禁止：无依据建议、替代医生诊断

## 2.3 执行层（Act）
- 支持动作：新增记录、更新记录、应用计划、应用清单、撤销最近动作
- 约束：按写库权限矩阵执行（Auto / Confirm / Block）
- 强制：每次动作都要有回执与审计日志

---

## 3. 默认执行原则

1. **单入口原则**：前端只保留“小人智能体入口”，不散落 AI 按钮。
2. **先生成后应用**：计划/采购先生成草案，用户确认后一键落库。
3. **先安全后效率**：医疗相关动作优先确认，风险触发优先阻断。
4. **先证据后结论**：所有建议需显示依据来源。
5. **先可撤销再自动化**：自动动作必须提供 undo token。

---

## 4. 对话到执行的标准链路

1) 用户输入自然语言
2) 意图识别 + Function Calling
3) 参数校验 + 风险判定
4) 执行（Auto / Confirm / Block）
5) 回执卡片（写入内容、影响范围、撤销入口）

---

## 5. 输出与回执规范

每次智能体返回至少包含：
- `summary`：本次做了什么
- `actions[]`：每个动作的状态与结果
- `riskLevel`：low/medium/high
- `undoToken`（若可撤销）
- `cards[]`（建议卡/计划卡/采购卡）

---

## 6. 医疗边界总则

AI 仅做生活管理建议（饮食/运动/作息），不做诊断结论。
触发高风险时：阻断普通建议并建议就医。

详细规则见：`04-risk-guardrails-v1.md`
